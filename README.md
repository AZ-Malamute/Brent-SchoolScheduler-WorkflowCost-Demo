# Brent School Scheduler Workflow-Cost Demo

**Brent School Scheduler Workflow-Cost Demo** is a small browser-based proof showing how broken workflows create hidden costs and how quickly a source-truth prototype can expose the problem.

## What it demonstrates

This demo models a school scheduling workflow with:

- classrooms
- teachers
- grade levels
- course schedules
- transcript / graduation checks
- room conflicts
- teacher conflicts
- source-truth JSON packet export
- workflow-cost modeling

## Why it exists

The point is not that this is a finished school scheduling product.

The point is that broken workflows are expensive. If a company pays for AI tools but developers run out of tokens, lose context, repeat work, or spend time fighting the system, the subscription cost is only a small part of the real cost.

At $60/month per developer:

- 100 developers = $72,000/year
- 200 developers = $144,000/year
- 300 developers = $216,000/year
- 500 developers = $360,000/year

If 200 developers lose only 30 minutes/day at a $100/hour loaded cost, that is approximately:

**$2.6M/year in lost workflow time**

## Core line

You are not just running out of tokens.  
You are running out of workflow.

## Source-truth principle

The workflow is the evidence.
